# Product Brief — Familia Ponquesito

## Qué es esto

Este repositorio construye un único producto para **Familia Ponquesito**, un
negocio real (no un caso ficticio ni un ejercicio de práctica), en el marco
del **Platzi Vibe Coding Challenge**.

El challenge se compone de una serie de retos consecutivos. El producto de
este repositorio **evoluciona reto a reto**: no se planifica de antemano la
versión final del sistema, sino que cada reto añade o ajusta capacidades
sobre lo ya construido. Evitamos diseñar hoy para necesidades que todavía no
conocemos.

## Objetivo inmediato

El objetivo actual y prioritario es **cumplir y destacar en el Reto 1**
(ver `docs/challenge-1.md`), no construir un sistema empresarial completo
para Familia Ponquesito. Cualquier decisión de alcance o arquitectura debe
evaluarse primero contra esa meta: ¿esto ayuda a ganar el reto actual?

## Qué NO es este documento

Este brief no define los requerimientos definitivos ni el roadmap completo
del negocio. Las necesidades reales y duraderas de Familia Ponquesito
(qué debe hacer el sistema a largo plazo, qué integraciones necesita, cómo
se mantiene la base de conocimiento, etc.) se evaluarán **después de que
termine la competencia**, con más información y sin la presión de tiempo del
challenge.

## Principios que se mantienen entre retos

- El negocio es real: no se inventan datos, productos, políticas ni tono de
  marca que no hayan sido provistos.
- Cada reto debe poder evaluarse de forma independiente (ver
  `docs/challenge-1.md` para los criterios del primero).
- Se prioriza simplicidad y una experiencia de usuario clara sobre
  arquitectura sofisticada.
- Las decisiones importantes de alcance o estructura se documentan a medida
  que se toman.
